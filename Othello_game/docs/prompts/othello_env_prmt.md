# 要件
* クラス名: `OthelloEngine`
* 以下の定数をクラス外またはクラス変数として定義すること。
  * `EMPTY = 0`
  * `PLAYER_BLACK = 1`
  * `PLAYER_WHITE = -1`
* 以下の初期盤面生成関数を**そのまま**クラスのメソッドとして組み込むこと。
```python
def get_initial_board(self) -> List[List[int]]:
        """8x8の初期盤面を生成します。

        Returns:
            List[List[int]]: 0=空, 1=黒, -1=白で構成される8x8の2次元配列
        """
        board = [[self.EMPTY] * 8 for _ in range(8)]
        board[3][3], board[4][4] = self.PLAYER_WHITE, self.PLAYER_WHITE
        board[3][4], board[4][3] = self.PLAYER_BLACK, self.PLAYER_BLACK
        return board

```

* 処理内容: 以下の機能をクラスのメソッドとして実装してください。
1. 手の合法性判定 (`is_valid_move`)
2. 石を置く＆裏返す処理 (`apply_move` または `step`)
3. パスの判定・手番の交代 (`has_valid_moves`, `change_turn`)
4. 勝敗・スコア判定 (`calculate_winner`)


# 入力 (Input)

* `board`: `List[List[int]]`。8x8の2次元配列。
* `player`: `int`。現在の手番（1 または -1）。
* `row`, `col`: `int`。石を置く座標（0〜7）。

# 出力 (Output)

* 各メソッドの戻り値は、極力標準のPython型（`bool`, `int`, `List[List[int]]`, `Dict`など）を使用し、APIとしてそのままJSONにシリアライズ可能な形式にすること。

# 制約事項・エッジケース (Constraints)

1. **型定義の徹底**: Python 3.12以上の型ヒント（`List`, `Tuple`, `Optional`, `Dict` など）をすべての引数と戻り値に厳格に使用してください。
2. **I/Oの標準化**: 内部計算で `numpy` を使用して最適化を図るのは自由ですが、クラスのパブリックメソッドの入出力（特に盤面データ）は必ず `List[List[int]]` に変換・統一してください。APIのJSON化でエラーが起きないようにするためです。
3. **盤面のバリデーション**: 外部から渡される盤面データが「要素数8のリストが8行連なる2次元配列」であること、かつ「0, 1, -1以外の不正な値が含まれていないこと」を担保する内部検証ロジックを設けてください。
4. **エッジケースと例外処理**:
* `row` や `col` が0〜7の範囲外（Out of bounds）になるケース。
* すでに石が置かれているマスに対する操作。
* これらに対しては、システムがクラッシュしないよう安全に `False` を返すか、意図的な不正リクエストとして明確な `ValueError` を送出するようエラーハンドリングを徹底してください。
